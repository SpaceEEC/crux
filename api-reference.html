    Crux vdev API Reference

    Modules

    Crux.Base

    Crux “Base” module providing an example implementation of Crux.Gateway, Crux.Rest, and Crux.Cache while using Crux.Structs

    Crux.Base.Consumer

    Handles consuming and processing of events received from the gateway.
    To consume those processed events subscribe with a consumer to a producer.
    You can fetch said producers via Crux.Base.producers/0

    Crux.Base.ConsumerSupervisor

    Supervises a consumer, for example, a module using Crux.Base.TaskConsumer

    Crux.Base.TaskConsumer

    Provides a __using__ macro to inject two functions to simplify consuming of gateway events

    Crux.Cache

    Behaviour all caches must implement. (Looking at custom ones you may want to write)

    Crux.Cache.Base

    Base cache utilising ETS tables, :ets.new/2

    Crux.Cache.Channel

    Default Crux.Structs.Channel cache

    Crux.Gateway

    Main entry point to start the Gateway connection

    Crux.Gateway.Application

    Root for the supervision tree

    Crux.Gateway.Command

    Builds Gateway Commands.
    Note: Only the sent ones can be found here

    Crux.Gateway.Connection

    Handles the actual connection to Discord

    Crux.Gateway.Connection.Producer

    Handles dispatching of packets received from the gateway

    Crux.Gateway.Connection.RateLimiter

    Handles Rate Limiting for the gateway

    Crux.Gateway.IdentifyLimiter

    Handles the Identifying rate limit of 1 per 5 seconds

    Crux.Gateway.Util

    Collection of util functions

    Crux.Rest

    Collection of Rest functions

    Crux.Rest.CDN

    Functions to generate cdn urls pointing to avatars, icons, etc

    Crux.Rest.Endpoints

    Endpoints being used by the Crux.Rest module, you do not need to worry about it

    Crux.Rest.Handler

    Handles per route rate limiting

    Crux.Rest.Handler.Global

    Handles global rate limits and the average time offset to discord

    Crux.Rest.Util
